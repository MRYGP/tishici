# 系统指令说明

这个目录包含4个核心提示词，支撑整个商业案例学习系统。

---

## 🤖 四大提示词

### 1. Claude-案例拆解提示词v1.0.md ⭐ 核心
**作用**：系统化拆解单个商业案例

**三大工作模式**：
- ⚡ 快速拆解（<30分钟）：快速建立认知框架
- 🎯 深度拆解（1-2小时）：七步完整拆解
- 🔬 跨案例对比（2-3小时）：对比2-3个相似案例

**七步拆解法**：
1. 时空还原（Context）
2. 问题诊断（Problem）
3. 决策拆解（Decision）
4. 执行拆解（Execution）
5. 模型提炼（Model）
6. 成败分析（Outcome）
7. 迁移应用（Application）

---

### 2. Claude-模型提炼提示词v1.0.md
**作用**：从案例中提炼可复用的商业模型

**三大工作模式**：
- ⚡ 单案例模型提炼（<45分钟）
- 🎯 多案例模型融合（1-1.5小时）
- 🔬 模型对比验证（1.5-2小时）

**五步提炼法**：
1. 案例共性识别
2. 变量体系构建
3. 因果链条梳理
4. 边界条件界定
5. 决策框架转化

---

### 3. Claude-洞察归纳提示词v1.0.md
**作用**：从多个案例中挖掘跨案例的深层规律

**三大工作模式**：
- ⚡ 单主题洞察（<1小时）
- 🎯 跨主题洞察（1.5-2小时）
- 🔬 趋势预测洞察（2-3小时）

**六步归纳法**：
1. 主题聚类
2. 对比分析
3. 规律提炼
4. 反例验证
5. 预测验证
6. 应用框架

---

### 4. Cursor-文档生成指令v1.0.md
**作用**：接收Claude的分析结果，生成标准化Markdown文档

**核心功能**：
- 结构化转换（分析→文档）
- 自动同步（更新索引、交叉引用）
- 质量保障（格式、内容、链接）

**三类模板**：
- 案例拆解文档模板
- 模型提炼文档模板
- 洞察归纳文档模板

---

## 🔄 Claude-Cursor协作流程

```
用户提供原始信息
         ↓
Claude深度分析推理（使用对应提示词）
         ↓
Claude输出结构化要点 + Cursor指令
         ↓
用户复制给Cursor执行
         ↓
Cursor生成标准文档 + 自动同步
         ↓
保存到GitHub对应目录
```

## 💡 最佳实践

### 新手建议
1. 从**快速拆解模式**开始，建立基本认知
2. 选择熟悉的行业/产品作为第一个案例
3. 严格遵循提示词的工作流程
4. 完成后复盘：哪些步骤卡住了？如何改进？

### 进阶技巧
1. 同一案例先快速拆解，再深度拆解
2. 拆解3-5个相似案例后，再做模型提炼
3. 定期（每月）做跨案例洞察归纳
4. 建立个人的"案例-模型-洞察"知识图谱

## 📋 质量标准

### 案例拆解质量
- [ ] 时空背景还原清晰？
- [ ] 关键决策都拆解了？
- [ ] 有数据支撑？
- [ ] 提炼了可迁移的框架？

### 模型提炼质量
- [ ] 变量体系完整？
- [ ] 因果链条清晰？
- [ ] 适用边界明确？
- [ ] 有决策应用建议？

### 洞察归纳质量
- [ ] 有足够案例支撑（3+）？
- [ ] 规律非显而易见？
- [ ] 经过反例验证？
- [ ] 可转化为决策工具？

---

## 🚫 常见错误

1. **案例拆解**：只罗列事实，没有分析"为什么"
2. **模型提炼**：理论堆砌，没有结合案例
3. **洞察归纳**：凭空想象，没有案例支撑
4. **文档生成**：格式混乱，不遵循模板
