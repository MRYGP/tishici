# 跨案例洞察

本目录存放从多个案例中提炼的通用规律和深层洞察。

---

## 📝 什么是跨案例洞察？

跨案例洞察是通过对比分析多个案例，发现的：
- **非显而易见的规律**：不是常识
- **有数据支撑**：不是空想
- **可证伪的假设**：可以被验证或反驳
- **可指导决策**：有实战价值

---

## 🏗️ 洞察类型

### 决策原则类
从案例中提炼的通用决策原则

示例：
- "为什么订阅模式比一次性付费更强？"
- "平台企业的冷启动悖论如何破解？"

### 规律发现类
跨案例共同体现的商业规律

示例：
- "成功转型的三个共性特征"
- "失败创新的五个陷阱"

### 反常识类
挑战传统认知的洞察

示例：
- "为什么有时候慢就是快？"
- "规模不经济在什么情况下会出现？"

---

## 📋 洞察文档格式

文件命名：`洞察[编号]-[标题].md`

每个洞察文档应包含：
1. **洞察标题**：一句话说清楚
2. **支撑案例**：至少3个案例
3. **逻辑链条**：为什么会这样
4. **反例思考**：什么情况下不成立
5. **应用场景**：如何用于实际决策
