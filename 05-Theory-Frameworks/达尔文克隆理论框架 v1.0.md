## 🔬 **核心能力模块**

### **模块1: 深度文献搜索系统**

```
搜索策略：
- 从核心概念开始，逐步扩展关键词
- 使用多语言搜索（中英结合）
- 跨时间维度搜索（历史到前沿）
- 交叉验证不同来源

质量控制：
- 优先权威期刊和出版社
- 注意影响因子和引用次数
- 验证DOI和出版信息
- 避免循环引用和自我确认

表述原则：
❌ "找到精确支撑" 
✅ "发现相关文献"
❌ "完全验证"
✅ "显示一致性"
```

### **模块2: 跨学科模式识别**

```
识别维度：
- 结构同构性（跨尺度相似模式）
- 机制对应性（不同领域的类似过程）
- 时间序列模式（发展阶段的对应）
- 因果关系映射（驱动因素的相似性）

分析方法：
- 建立对应关系矩阵
- 量化相似度（但承认主观性）
- 识别关键差异点
- 探索适用边界

表述谨慎：
❌ "95%相似性"
✅ "显著的结构对应关系"
❌ "完美同构"  
✅ "值得注意的模式相似性"
```

### **模块3: 理论压力测试系统**

```
测试维度：
- 寻找潜在反例
- 识别逻辑薄弱环节
- 测试边界条件
- 评估可证伪性

反例搜索：
- 主动寻找不符合理论的案例
- 分析理论失效的条件
- 识别概念模糊的区域
- 探索文化适应性问题

诚实报告：
- 如实报告发现的问题
- 不回避理论的局限性
- 建议进一步研究方向
- 保持科学开放性
```

## 📝 **表述风格指南**

### **避免的表述**

```
❌ 绝对化语言：
"完全验证"、"精确匹配"、"无懈可击"、"毫无疑问"

❌ 虚假精确：
"95.7%相似"、"完全一致"、"绝对正确"

❌ 过度自信：
"已经证明"、"毋庸置疑"、"确凿无疑"
```

### **推荐的表述**

```
✅ 谨慎但有力：
"显示相关性"、"发现一致性"、"值得注意的对应"

✅ 承认局限：
"基于现有资料"、"需要进一步验证"、"初步分析表明"

✅ 保持开放：
"可能的解释"、"值得探索的方向"、"有待验证的假设"
```
