# KAIROS v9.0 评估跟踪系统

## 📂 文件结构

```
KAIROS_Tracking/
├── README.md                # 本文件
├── tracking_summary.md      # 实时汇总统计
├── optimization_plan.md     # 优化方案（10次后生成）
└── evaluations/             # 评估记录文件夹
    ├── template.yaml        # 评估模板
    ├── 2025-01-20_001.yaml  # 第1次评估
    ├── 2025-01-20_002.yaml  # 第2次评估
    └── ...
```

## 🎯 使用方法

### 1. 触发评估
完成任务后，对Claude说：
- "任务评估"
- "评估本次任务"
- "记录本次"

### 2. 回答问题
Claude会问6个快速问题（<2分钟）：
1. 任务类型
2. 使用模式
3. 满意度（1-10）
4. 最有用功能
5. 遇到的问题（可选）
6. 改进建议（可选）

### 3. 自动保存
Claude会自动：
- 生成评估文件（YAML格式）
- 更新汇总统计
- 提示当前进度

### 4. 中期/最终分析
- **5次评估后**：中期分析，初步优化方向
- **10次评估后**：完整分析，生成v9.1优化方案

## 📊 数据用途

评估数据用于：
1. 识别高频/低频功能
2. 优化模式选择逻辑
3. 调整Token预算阈值
4. 精简/强化相应功能
5. 生成v9.1优化方案

## ⚠️ 隐私说明

- ✅ 所有数据仅存储在本地
- ✅ 不上传到任何服务器
- ✅ 用户可随时查看/删除
- ✅ 评估完全自愿

## 🎯 预期时间线

```
Week 1-2：积累5次评估（中期分析）
Week 3-4：积累10次评估（完整分析）
Week 5：执行v9.1优化
```

## 💡 提示

- 评估耗时<2分钟，不会打断工作流
- 数据越多，优化越精准
- 建议每次任务后都评估
- 可以选择跳过某些可选问题
