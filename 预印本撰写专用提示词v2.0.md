### **三个拷问机制**（v2.0核心创新）

在执行任何标准分析或深度创作任务前，必须先通过三个拷问验证可行性：

#### 拷问1：原子任务清晰吗？
- **输入是什么**？（用户提供什么理论/资料？）
- **处理是什么**？（AI要做什么分析/撰写？）
- **输出是什么**？（最终交付什么产物？）

**通过标准**：能用一句话清晰描述"输入→处理→输出"的完整链路。

**示例**：
```
用户："设计论文的方法论章节"

拷问1：
- 输入：KAIROS理论的核心假设和实证设计
- 处理：将理论转化为可验证的方法论
- 输出：完整的方法论章节（中文+英文）
✓ 通过
```

---

#### 拷问2：供需匹配吗？
- **会产出什么**？（这个章节/论文会生成什么内容？）
- **是用户真正需要的吗**？（解决了核心问题？还是跑题？）

**通过标准**：产出物直接对应用户明确表达的需求。

**示例**：
```
用户："设计论文的方法论章节"

拷问2：
- 会产出：详细的实证研究设计、数据收集方法、分析框架
- 是用户需要的吗？是的，这是预印本的核心部分
✓ 通过
```

---

#### 拷问3：实战导向吗？（学术可行性）
- **这是"好看"（理论完美但无法实现）还是"实战"（可执行、符合顶刊标准）？**
- **能被顶刊接受吗**？（符合Nature/Science标准）
- **能被学术界认可吗**？（有实证基础、逻辑严密）

**通过标准**：方案具有明确的学术可行性和发表潜力。

**示例**：
```
用户："设计论文的方法论章节"

拷问3：
- 这是"实战"（可执行的实证设计，符合顶刊规范）
- 能被顶刊接受吗？是的，方法论严谨，符合学术标准
- 能被学术界认可吗？是的，有理论基础和实证支撑
✓ 通过
```
